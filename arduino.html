Assignment 3 c

In deze opdrachten moesten we ervoor zorgen dat de groene led twee keer zo snel knipperde als de gele led. Voor deze opdracht hadden we al wat informatie gekregen. Dit was een afbeelding waarin stond hoe we de ledjes moesten aansluiten en een gegeven code. De opdracht was om die code dus zo aan te passen dat de groene led sneller knipperde als de gele led.

In de code is te zien dat we beide ledjes aanroepen met de pinMode. Nummer 12 en 13 zijn de nummers waarmee de ledjes verbonden zijn aan het arduino bord. Daarna is in de code te zien dat er een loop word gemaakt wat ervoor zorgt dat de ledjes de functie blijven uitvoeren. De HIGH en LOW vertellen de ledjes wanneer ze aan (high) en uit (low) moeten zijn. Door het delay aan te passen naar 500 veranderd de snelheid en zorgt ervoor dat de gele en groene led van elkaar verschillen.

Assignment 4 c

Voor deze opdracht moesten we de code zo aanpassen dat de eene led langzaam uit gaat en de ander langzaam aan. Dit moest tegelijkertijd gebeuren. Het geen wat aan de code is toegevoegd is de analogWrite van de yellow led pin. Hierin word aangegeven dat de yellow led pin een licht sterkte heeft van 255 min de brightness. De brightness is op dat moment de helft van de groene led. De groene led is namelijk al eerder opgeroepen.

Assignment 6 c

In deze opdracht kregen we een potmeter waarmee we de opdracht moesten uitvoeren. Een potmeter is een klein ding waar je aan kan draaien. De opdracht was dat wanneer je aan de potmeter draait de eene led langzaam aangaat, terwijl de andere led langzaam uitgaat. In de code hebben we een map toegevoegd. Die ervoor zorgt dat de waarde van het licht van 0 naar 1024 naar 0 naar 255 gaat. Om ze niet beide hetzelfde te laten doen hebben we de code bij de groene led aangepast dat de waarde van het licht de sensor value van de 255 af haalt. Op de manier veranderen ze niet tegelijk wanneer er aan de potmeter gedraaid wordt.

Assignment 7

De opdracht was om de ledjes uit te laten gaan wanneer er geen licht op de LDR kwam. De LDR is een licht sensor. In de code is te zien dat de waarde van de sensor, de sensor pin (LDR), het groene ledje en het gele ledje worden aangeroepen. Daarna word er gezegd dat de sensor pin informatie moet krijgen en dit moet uitvoeren naar de twee ledjes. In de loop word doorgegeven dat wat er gelezen wordt uit de pin word doorgevoerd als sensorValue. Er word in de loop gebruik gemaakt van een map. De map zorgt ervoor dat de waardes die werden aangegeven niet buiten de 0 tot 255 mogen valen. De waardes die eerst werden aangegeven lagen namelijk tussen de 200 en 830. Uiteindelijk word de value geprint en worden de waardes doorgevoerd naar de ledjes.

Assignment 10 f

De bedoeling van deze opdracht was dat wanneer je een van de twee knopjes ingedrukt hield, het ledje ging branden. Hierbij maakt het niet uit welke van de twee knopjes. Maar wanneer je ze beiden ingedrukt hield het ledje niet ging branden. In de code staat in de void setup dat de led pin out put moet geven (dus moet gaan branden) en dat de buttons beide informatie moeten krijgen (wanneer er op gedrukt word). De loop zorgt ervoor dat het uiteindelijk werkt. In de loop staat een zin met buttonPinL (de linker knop) en dan een (^) teken met als volgt buttonPinR (de rechter knop). Het teken (^) beter het een of het ander. Dus op de rechter of de linker knop moet worden ingedrukt en dan geeft de led een output HIGH, dus gaat het ledje aan). Wanneer dit dus niet zo is (dus beide knoppen worden ingedrukt), dan gaat het ledje niet aan.
